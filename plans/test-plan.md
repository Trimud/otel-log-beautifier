## Key Interactions to Verify
- Open beautified terminal, run a service that outputs OTel JSON — should see colorized output
- Run `ls`, `git status`, other normal commands — output unchanged
- Run `vim` or `less` — alt screen detection bypasses parser, no corruption
- Toggle beautification off — see raw JSON, toggle on — formatted again
- Set level filter to ERROR — only error lines shown, status line shows suppressed count
- Click a file path in a stack trace — opens file in editor at correct line
- Ctrl+C in terminal — signal forwarded to child process

## Edge Cases
- JSON line wrapped in ANSI color codes (some logging libs pre-colorize)
- Extremely long JSON line (>64KB) — should passthrough without detection
- Binary output from `cat /dev/urandom | head -c 100` — should passthrough
- pino + bunyan format overlap (same line matches both) — priority chain resolves correctly
- Resource block identical to previous line — suppressed (no duplicate resource output)
- Empty terminal (level filter active, no matching logs) — status line shows suppressed count
- node-pty unavailable — falls back to child_process.spawn with warning

## Critical Paths
- PTY bridge spawns shell and proxies I/O without data loss — this is the foundation
- Alt screen buffer detection (ESC[?1049h, ESC[?47h, ESC[?1047h) — prevents interactive program corruption
- JSON detection accuracy — no false positives (non-JSON classified as JSON), no false negatives (JSON missed)
- Extension deactivation kills all child processes gracefully
